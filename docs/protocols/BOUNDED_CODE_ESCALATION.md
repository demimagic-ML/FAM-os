# Bounded economical-to-strong code escalation

Code execution starts with the packaged economical expert. A failed verifier
result can enter escalation only after an atomic plan-global reservation. The
reservation charges worst-case output tokens and wall time before inference;
denial prevents the delegate call. Escalation and escalation-repair attempts
share the ledger with other repairs.

Repair prompts disclose at most 4,000 failure characters plus digest-bound
verifier-owned tests/examples. The original prompt and `stable-toposort-v2`
acceptance remain unchanged. Feedback explains failure but has no authority to
remove assertions or release a candidate.

Canonical full-workstation traces under `artifacts/expert_fabric/phase9.3/`
prove:

- Qwen 2.5 Coder 7B fails, Gemma 4 26B escalates and passes initially.
- Qwen 2.5 Coder 7B fails, Laguna XS.2 escalates, fails, consumes one bounded
  repair reservation, and passes.
- Both strong identities and weight digests match installed runtime bindings.
- Each trace binds raw report, trusted tests, repair examples, package artifact,
  attempt sequence, acceptance ID, and global budget snapshot.
