# Expert Factory lifecycle

Phase 13 starts only from independently verified, digest-bound failure traces. Repeated exact capability/requirement clusters create proposals but never authorize training. Teacher datasets, distillation, adapters, held-out evaluation, and shared interfaces are explicit contracts.

Routing, complexity, and the stable-order specialist are actually trained deterministic token classifiers. The hardware objective penalizes activated parameters, bytes moved, latency, and joules. Quantized variants must retain calibration quality. Packages are atomically published with Ed25519 signatures, verified before installation, continuously gated against quality/latency/energy/acceptance baselines, and removable through retirement.

The canonical lifecycle discovered the stable-order gap, trained three experts, applied the hardware objective, generated a calibrated int4 variant, signed and published it, verified installation, selected and verified the specialist, then retired it with no installed artifact remaining.
